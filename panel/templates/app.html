Подключить новую ноду

Сценарий: нажимаешь одну из кнопок выше, указываешь имя, публичный хост и название в подписке, и получаешь одну bash-команду. Запускаешь её на свежем Ubuntu — скрипт установит xray + агента, сам зарегистрируется в этой панели, панель сгенерирует Reality-ключи и запушит конфиг.

⚡ Нода авто-балансировки — то же самое, но сервер сразу попадает в пул. В подписке такие ноды объединяются в группу «⚡ Auto (Pool)», и клиент (Hiddify / sing-box / Clash / Karing / Happ) сам выбирает самый быстрый узел пула.

🎯 Балансер-нода — отдельный сервер, который получает один vless-ключ и сам внутри выбирает самый быстрый upstream из пула (xray observatory + leastPing). Этот ключ работает в любом клиенте (v2rayN, Shadowrocket, Hiddify). Чтобы ключ был полезен, в пуле (⚡) должно быть хотя бы 2 ноды.

🇷🇺→🌍 Нода обхода (whitelist) — связка из двух серверов. Сначала ставишь обычную ⚡ ноду за границей (Германия / Финляндия / Нидерланды) — это будет «выход». Потом запускаешь эту кнопку, чтобы поставить РФ-фронт на whitelisted IP и сразу прицепить его к иностранному выходу. Пользователи коннектятся именно на РФ-фронт (его IP не дросселируется ТСПУ); xray на фронте пересылает весь трафик одним VLESS+Reality-прыжком в иностранную ноду, которая уже выпускает его в интернет. В подписке/боте видно только РФ-фронт — иностранный выход спрятан.

Подписки

Подписка — это одна ссылка, в которой клиент видит все выбранные VLESS-ключи из нескольких серверов сразу. Совместимо с v2rayN, Hiddify, Streisand, Happ и другими клиентами с полем «Subscription URL».

API-токены

API-токены используются ботами и скриптами для доступа к панели без session-cookie. Передавай в заголовке Authorization: Bearer <token>. Токен виден ОДИН раз в момент создания — сохрани его сразу.

Глобальный домен подписок

Используется для всех ботов, у которых не задан индивидуальный домен. Можно с портом, если reverse-proxy не на :443 (там Reality): https://sub.example.com:4443.

Бэкенд авто-провижна: . После сохранения панель автоматически выпустит Let's Encrypt сертификат и сгенерирует vhost.

Telegram-боты

Боты запускаются прямо внутри процесса панели (отдельный systemd-юнит не нужен). Каждому твоему пользователю в боте панель автоматически выдаёт отдельный vless-ключ. Если с ключа идёт больше устройств, чем лимит бота, тебе в chat владельца прилетит уведомление с кнопками «Забанить» / «Игнорировать».

💳 Оплата

Здесь настраиваются тарифы и подключаются провайдеры платежей: Telegram Stars, CryptoBot, FreeKassa. Когда пользователь нажимает «🛒 Купить подписку» в боте, он видит только включённые тарифы и включённые провайдеры. После успешной оплаты срок действия ключа продлевается автоматически (или, если ключ уже истёк — сбрасывается с «сегодня + N дней»).

Обновить все ноды

Каждая нода получит команду xnpanel update --force: агент подтянет последний коммит из , перезапустит xray-agent.service (на хосте панели — ещё и xray-panel.service). Reality-ключи и UUID-ы клиентов не трогаются — пользователи продолжат коннектиться без ре-импорта ссылок. Хост панели обновляется в самом конце списка, чтобы не убить этот запрос на середине.

Проверь доступность xray-agent.service и сетевую связность. Хост панели в списке — соединение с панелью на ~10 секунд могло моргнуть, это нормально.

Установочная команда готова

⚡ Попадёт в primary-бакет авто-балансировки (легаси «in_pool»=true). В подписке — в группе «⚡ Auto (Pool)».

🛡 Попадёт в fallback-бакет. Клиент переключается сюда, если все ⚡ primary недоступны (белые списки в регионе включились). Каждые сек sing-box пробует primary вернуться.

🎯 Это отдельная нода-балансер. После установки она получит один vless-ключ; xray внутри сам будет выбирать самый быстрый upstream из пула по leastPing. В пуле (⚡) должно быть минимум 2 standalone-ноды — иначе балансеру некого выбирать.

Пока enrollment «pending», любой, у кого есть команда, может добавить ноду под этим именем. Токен одноразовый — после успеха он становится невалидным.

Скопируй команду и запусти её на новом Ubuntu 24.04-сервере под root. Сервер сам появится в панели через 1–2 минуты.
