Подключить новую ноду

Сценарий: ты нажимаешь «Новая enrollment-команда», указываешь имя и публичный хост будущей ноды, и получаешь одну bash-команду. Запускаешь её на свежем Ubuntu — скрипт установит xray + агента, сам зарегистрируется в этой панели, панель сгенерирует Reality-ключи и запушит конфиг. Никаких ручных токенов.

Подписки

Подписка — это одна ссылка, в которой клиент видит все выбранные VLESS-ключи из нескольких серверов сразу. Совместимо с v2rayN, Hiddify, Streisand, Happ и другими клиентами с полем «Subscription URL».

API-токены

API-токены используются ботами и скриптами для доступа к панели без session-cookie. Передавай в заголовке Authorization: Bearer <token>. Токен виден ОДИН раз в момент создания — сохрани его сразу.

Telegram-боты

Боты запускаются прямо внутри процесса панели (отдельный systemd-юнит не нужен). Каждому твоему пользователю в боте панель автоматически выдаёт отдельный vless-ключ. Если с ключа идёт больше устройств, чем лимит бота, тебе в chat владельца прилетит уведомление с кнопками «Забанить» / «Игнорировать».

Telegram-уведомления

Панель пушит события в Telegram: создание/удаление ключей, авто-отключение по лимиту, ребут сервера. Создай бота через @BotFather и укажи chat id (узнать — @userinfobot).

Добавить сервер (вручную)

Используй, если агент уже установлен на ноде и ты знаешь его URL + токен. Для пустого Ubuntu-сервера проще — вкладка «Новая нода» с готовой bash-командой.

Новая enrollment-команда Установочная команда готова

Пока enrollment «pending», любой, у кого есть команда, может добавить ноду под этим именем. Токен одноразовый — после успеха он становится невалидным.

Скопируй команду и запусти её на новом Ubuntu 24.04-сервере под root. Сервер сам появится в панели через 1–2 минуты.
